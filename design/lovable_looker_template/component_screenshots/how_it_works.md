**How the getmock.rittmananalytics.com URL works**

1. Loveable has a rudimentary API, with the first example of it being their [Build with URL](https://docs.lovable.dev/integrations/build-with-url) endpoint. For example, you could use URL:

```
https://lovable.dev/?autosubmit=true#prompt=Create%20a%20todo%20app
```

and it would take you to the Loveable.dev website, make you login in or signup, prompt you for a workspace to put the new site in, and then create you a new website/app based on your `Create a todo app` prompt.

2. The prompt though can be up to 50k characters in-length, and can include the parameter `images` which can contain up-to URL-encoded image URLs as references. Image URLs must be publicly accessible. Supported formats: JPEG, PNG, WebP

3. Previously, I asked loveable.dev (via a prompt) to detail for me the look-and-feel it had assembled for our Looker mockup sites, the details of which I then copied into [prompt_look_and_feel.md](design/loveable_looker_template/prompt_look_and_feel.md) for reference.

4. As part of the prompting session, I also asked loveable.dev what screenshots I should take to accompany these details if I wanted loveable to recreate that same look-and-feel in a future site. I took those screenshots and uploaded them [here](design/loveable_looker_template/component_screenshots).

5. I then used the [Loveable Link Generator](https://docs.lovable.dev/integrations/build-with-url#lovable-link-generator) feature on their website to assemble a template URL for creating new Looker mockup apps, by pasting the [prompt_look_and_feel.md](design/loveable_looker_template/prompt_look_and_feel.md) and selecting the URLs of the component screenshots, and added an extra line to the start of the prompt with placeholder text to hold the use-case name, e.g.
